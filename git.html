Qu'est ce que GIT

Qui est un outil de versionning qui permet d'avoir une traçabilité et un historique de tous les fichiers du/des projet. Il permet également de travailler en équipe (envoyer modifications à nos collègues et récupérer les modifications de collègues. Dans l'historique des fichiers GIt nous donne toujours 3 infos : qui quand pourquoi ?

Les commandes de base

Initialiser un projet:
git init
Voir le statut des fichiers
git status
Ajouter des Fichiers
git add'fichier'
Ajouter tous les Fichier:
git commit
commit(enregistrer) les fichiers ajoutés
git commit
ATTENTION Un commit necessite toujours un message
commit avec message:
git commit -m "message"
commit avec add + message(fonctionne sur les fichiers déja add précédement ):
git commit -a -m "message"

Correction des fichiers

voir les différents commit:
git log
Corriger les modification sur un ficihier (pas encore add):
git checkout 'fichier'
Supprime un add:
git reset 'fichier'
git reset .
Supprimer le dernier commit:
git reset HEAD^
Supprimer les deux derniers commit :
git reset HEAD ^^

La notion de branche

Une branche est une copie du projet , ELle permet de dév elopper en parralelle de nouvelles fonctionnalités. Lorsqu'on considère lesnouveautés opérationnelle, on peut les rappatrier sur la version principale (master)

Voir les branches:
git branch
Crée une nouvelle branche:
git branche 'nom'
Se placer sur une branche :
git checkout 'nom'
voir les différences entre 2 branches:
git diff 'nom'.. master
Apporter les modif effectués sur une branche dans master(il faut etre sur master)
git checkout master
git fetch.'nom
Le fetch va literallement remplacer master par la branche. Il s agit donc d une grosse modification dans le projet (ex:version 2
